================================================================================
FOOD REDISTRIBUTION APP - PROJECT STRUCTURE (simple guide)
================================================================================
After "git pull", run:  flutter pub get
Then run:  flutter run   (or open in Android Studio / VS Code and run)

--------------------------------------------------------------------------------
WHAT TO EDIT (your code lives here)
--------------------------------------------------------------------------------

  lib/
    main.dart                 --> App entry point
    config/                   --> Firestore schema, app config
    constants/                --> App-wide constants
    models/                   --> Data models (User, Donation, Request, etc.)
    providers/                --> State (auth, theme, donations, etc.)
    screens/                   --> All app screens by role
      auth/                    --> Login, signup, forgot password, onboarding
      admin/                   --> Admin dashboard, verify users
      donor/                   --> Donor dashboard, create donation, verification
      ngo/                     --> NGO dashboard, requests, documents, delivery
      volunteer/               --> Volunteer tasks, accept/reject, execution
      coordination/           --> Delivery coordination, issue reporting
      logistics/               --> Logistics dashboard
    services/                 --> API, Firebase, tracking, analytics
    utils/                    --> Router, theme, helpers
    widgets/                   --> Reusable UI (maps, loading, text fields)

  test/                        --> All tests (unit, widget, integration, tracking)
    unit/
    widget/
    integration/
    tracking/                  --> Delivery lifecycle and tracking tests

--------------------------------------------------------------------------------
FOLDERS AT APP ROOT (do not rename)
--------------------------------------------------------------------------------

  docs/                       --> Project docs (STRUCTURE.txt, README_FIRST, CLEAN_GIT_ONCE)
  scripts/                    --> Run tests, test_matching.dart, verify_schemas.js
  assets/                     --> Images and icons used by the app
    images/                   --> Add app images here
    icons/                    --> Add app icons here
    screenshots/               --> App screenshots (e.g. flutter_01.png - 04.png)
  _archive/                   --> Old/unused code kept for reference (e.g. landing_page_frontend)

--------------------------------------------------------------------------------
VERSION AND DEPENDENCIES (do not delete)
--------------------------------------------------------------------------------

  pubspec.yaml                --> App name, version, Dart/Flutter dependencies

--------------------------------------------------------------------------------
PLATFORM FOLDERS (only change if you need native Android/iOS/Web)
--------------------------------------------------------------------------------

  android/                    --> Android app (MainActivity, manifest, gradle)
  ios/                        --> iOS app
  web/                        --> Web (index.html, icons)
  linux/, macos/, windows/    --> Desktop (optional)

  Do NOT edit under android/app/build/ or android/.gradle/ - they are generated.

--------------------------------------------------------------------------------
CONFIG FILES (Firebase, rules - keep in repo)
--------------------------------------------------------------------------------

  firebase.json
  firestore.rules

--------------------------------------------------------------------------------
FILES NOT IN GIT (ignored - so no conflicts on git pull)
--------------------------------------------------------------------------------

  *.md                        --> Markdown docs (README.md is kept)
  build/                      --> Build output
  .dart_tool/                 --> Dart/Flutter tooling
  android/.gradle/
  android/local.properties
  android/app/build/
  ios/Pods/
  .idea/ , .vscode/           --> IDE settings
  google-services.json        --> Add locally for Firebase
  firebase_options.dart       --> Generated by FlutterFire
